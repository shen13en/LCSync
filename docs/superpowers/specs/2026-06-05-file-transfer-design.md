# LCSync 教师模式文件传输与作业管理功能设计

> 日期：2026-06-05
> 状态：草稿
> 版本：v1.0

## 1. 概述

在 LCSync 实验室中心联屏系统现有屏幕广播功能的基础上，增加教师模式下的文件共享和学生作业提交能力。

### 1.1 目标

- 教师可以上传文件到共享区，供学生浏览下载
- 学生可以实名提交作业文件给教师
- 教师可以查看和管理学生提交的作业
- 文件传输不影响视频广播的质量
- 保持现有技术栈（.NET Framework 4.7.2 + WPF + WebSocketSharp）

### 1.2 设计原则

- **与视频流隔离**：文件数据传输走独立通道，不挤占 WebSocket 视频广播带宽
- **尊重现有架构**：不引入第三方 HTTP 服务器库，不重写现有通信层
- **循序渐进**：第一版实现核心流程，大文件分块等优化放入后续迭代
- **配置灵活**：存储路径、带宽限制等参数可由教师自定义

## 2. 架构设计

### 2.1 总体架构

```
┌──────────────────────────────────┐      WebSocket :9456       ┌──────────────────────────────────┐
│          教师端                    │◄─────────────────────────►│          学生端                   │
│                                  │       视频广播 / 控制       │                                  │
│  ┌────────────────────────────┐  │                            │  ┌────────────────────────────┐  │
│  │  ScreenCaptureService      │  │                            │  │  SignalingClient           │  │
│  │  VideoEncoderService       │  │                            │  │  (视频接收 + 解码显示)     │  │
│  │  SignalingServer           │  │                            │  └────────────────────────────┘  │
│  └────────────────────────────┘  │                            │                                  │
│                                  │                            │                                  │
│  ┌────────────────────────────┐  │      HTTP :9457            │  ┌────────────────────────────┐  │
│  │  FileServerService         │◄──────────────────────────►│  │  HttpClient + 文件管理器      │  │
│  │  (HttpListener)            │  │                            │  │  (文件浏览/下载/上传)       │  │
│  └────────────────────────────┘  │                            │  └────────────────────────────┘  │
└──────────────────────────────────┘                            └──────────────────────────────────┘
```

- 教师端运行两个服务，端口相邻（9456 视频 + 9457 文件），互不干扰
- 文件相关的**通知消息**（新文件、新提交）走现有 WebSocket 通道，确保实时性
- 实际文件数据传输走 HTTP 通道，利用其内置的 Range 支持和流式处理能力

### 2.2 通知消息扩展

在现有的 `MessageType` 枚举中增加通知消息类型，**仅用于在 WebSocket 通道上发送轻量通知**：

| 消息类型 | 值 | 方向 | 用途 |
|----------|-----|------|------|
| `FileNotify` | 0x30 | 教师 → 学生 | 教师上传新共享文件时通知所有学生 |
| `SubmissionNotify` | 0x31 | 学生 → 教师 | 学生提交作业时通知教师 |

这些通知仅包含基本信息（如文件 ID、文件名），实际数据获取走 HTTP API。

## 3. 数据模型

### 3.1 FileItem（共享文件）

```csharp
public class FileItem
{
    public string Id { get; set; } = Guid.NewGuid().ToString("N");
    public string FileName { get; set; } = string.Empty;
    public long Size { get; set; }
    public DateTime UploadTime { get; set; }
    public int DownloadCount { get; set; }
}
```

### 3.2 SubmissionItem（作业提交记录）

```csharp
public class SubmissionItem
{
    public string Id { get; set; } = Guid.NewGuid().ToString("N");
    public string StudentName { get; set; } = string.Empty;
    public string StudentId { get; set; } = string.Empty;
    public string FileName { get; set; } = string.Empty;
    public long Size { get; set; }
    public DateTime SubmitTime { get; set; }
    public string StoragePath { get; set; } = string.Empty;  // 服务端实际存储路径
}
```

### 3.3 StorageConfig（配置模型）

```csharp
public class StorageConfig
{
    public string SharedDirectory { get; set; } = "";      // 共享文件目录
    public string SubmissionDirectory { get; set; } = "";   // 提交文件目录
    public long MaxBandwidthBytesPerSecond { get; set; } = 0; // 0 = 不限速
}
```

## 4. HTTP API 设计

FileServerService 基于 `System.Net.HttpListener` 实现，提供以下 REST 接口：

### 4.1 接口清单

| 方法 | 路径 | 用途 | 调用方 |
|------|------|------|--------|
| `GET` | `/api/student/info` | 获取学生身份标识 | 学生端 |
| `GET` | `/api/files` | 获取共享文件列表（JSON） | 学生端 |
| `GET` | `/api/files/{id}` | 下载共享文件（支持 Range） | 学生端 |
| `POST` | `/api/teacher/upload` | 教师上传共享文件 | 教师端 |
| `POST` | `/api/upload` | 学生提交作业（含姓名学号） | 学生端 |
| `GET` | `/api/submissions` | 获取所有提交记录（JSON） | 教师端 |
| `GET` | `/api/submissions/{id}` | 下载某份提交的作业 | 教师端 |
| `DELETE` | `/api/files/{id}` | 删除共享文件 | 教师端 |

### 4.2 关键接口详情

#### GET /api/files

返回教师共享区所有文件的 JSON 数组：

```json
[
  {
    "id": "abc123",
    "fileName": "第3章-操作系统.ppt",
    "size": 5242880,
    "uploadTime": "2026-06-05T14:30:00",
    "downloadCount": 12
  }
]
```

#### GET /api/files/{id}

下载文件。支持 HTTP Range 头实现断点续传。

- 不带 Range：返回完整文件（状态码 200）
- 带 Range：返回指定字节范围（状态码 206）

响应头包含 `Accept-Ranges: bytes`、`Content-Length`、`Content-Type`（基于扩展名自动检测）。

#### POST /api/upload

学生提交作业。使用 `multipart/form-data` 格式，包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `studentName` | string | 学生姓名（必填） |
| `studentId` | string | 学生学号（必填） |
| `file` | file | 作业文件 |

教师端收到后保存到提交目录，记录 SubmissionItem，并通过 WebSocket 广播 SubmissionNotify 通知教师端 UI 刷新。

#### POST /api/teacher/upload

教师上传共享文件。使用 `multipart/form-data` 格式：

| 字段 | 类型 | 说明 |
|------|------|------|
| `file` | file | 共享文件 |

教师端保存到共享目录，记录 FileItem，并通过 WebSocket 广播 FileNotify 通知所有已连接的学生。

#### GET /api/submissions

返回所有学生提交记录的 JSON 数组：

```json
[
  {
    "id": "sub001",
    "studentName": "张三",
    "studentId": "2024001",
    "fileName": "实验报告1.docx",
    "size": 2097152,
    "submitTime": "2026-06-05T14:35:00"
  }
]
```

### 4.3 错误处理

所有接口在出错时返回统一格式的 JSON 响应：

```json
{
  "error": "描述错误原因的消息"
}
```

HTTP 状态码：
- `200` — 成功
- `206` — 部分内容（Range 请求）
- `400` — 请求参数错误
- `404` — 文件不存在
- `413` — 文件超过大小限制
- `500` — 服务器内部错误

## 5. 带宽控制策略

### 5.1 令牌桶算法

在 FileServerService 中实现简单的令牌桶算法，限制文件下载速度：

- 每秒补充令牌（可用字节数），上限为 `MaxBytesPerSecond`
- 每次发送数据前检查可用令牌，不足则等待
- 默认值为 `0`（不限速），教师可在设置页面配置

### 5.2 带宽控制作用于

- **下载方向**（学生下载共享文件）受带宽限制
- **上传方向**（学生提交作业）不受限（上传通常远小于下载量，且频率低）
- 教师上传共享文件不受限

## 6. 存储管理

### 6.1 默认目录结构

```
%AppData%\LCSync\
├── Shared\           ← 教师上传的共享文件
│   ├── 第3章-操作系统.ppt
│   └── 实验指导书.pdf
├── Uploads\          ← 学生提交的作业文件
│   ├── 2024001_张三_实验报告1.docx
│   └── 2024002_李四_lab1.cpp
└── settings.json     ← 配置信息
```

### 6.2 配置文件

```json
{
  "sharedDirectory": "%AppData%\\LCSync\\Shared",
  "submissionDirectory": "%AppData%\\LCSync\\Uploads",
  "maxBandwidthBytesPerSecond": 0
}
```

教师在设置页面修改路径后实时生效，并持久化到 `settings.json`。

## 8. 实现方案

### 8.1 新增文件

| 文件路径 | 说明 |
|----------|------|
| `src/LCSync.App/Models/FileItem.cs` | 共享文件数据模型 |
| `src/LCSync.App/Models/SubmissionItem.cs` | 提交记录数据模型 |
| `src/LCSync.App/Models/StorageConfig.cs` | 存储配置模型 |
| `src/LCSync.App/Services/FileServerService.cs` | HTTP 文件服务（核心新增） |
| `src/LCSync.App/Utils/ConfigManager.cs` | 配置读写持久化工具 |

### 8.2 修改文件

| 文件路径 | 修改内容 |
|----------|----------|
| `Models/Messages.cs` | 增加 `FileNotify = 0x30`、`SubmissionNotify = 0x31` |
| `Services/SignalingServer.cs` | 增加 `BroadcastFileNotify` 和 `SendSubmissionNotify` 方法 |
| `Services/SignalingClient.cs` | 增加 `FileNotifyReceived` 和 `SubmissionNotifyReceived` 事件 |
| `ViewModels/TeacherViewModel.cs` | 增加文件共享、提交管理、设置相关的属性和命令 |
| `ViewModels/StudentViewModel.cs` | 增加文件浏览、下载、上传相关的属性和命令 |
| `Views/TeacherWindow.xaml` | 改为 TabControl 布局，新增 3 个标签页 |
| `Views/TeacherWindow.xaml.cs` | 新增标签页相关的代码后置逻辑 |
| `Views/StudentWindow.xaml` | 改为 TabControl 布局，新增 2 个标签页，底部状态栏不显示在线人数 |
| `Views/StudentWindow.xaml.cs` | 新增标签页相关的代码后置逻辑 |

### 8.3 不需要修改的文件

- `ScreenCaptureService.cs` — 屏幕捕获逻辑不变
- `VideoEncoderService.cs` — 视频编码逻辑不变
- `LightweightVideoEncoder.cs` — 编码实现不变
- `ModeSelectionWindow.xaml` — 模式选择窗口不变
- `ModeSelectionViewModel.cs` — 模式选择逻辑不变
- `FirewallUtils.cs` — 新增防火墙上会处理 9457 端口

## 9. 实施步骤

### 阶段一：基础设施（后端）

## 10. 后续迭代

- **大文件分块上传**：>100MB 文件的分块上传与合并
- **批量下载**：学生端多选文件批量下载
- **作业截止时间**：教师设置截止时间，过期禁止提交
- **文件夹共享**：支持上传整个目录结构
- **下载限速动态调整**：根据当前视频广播码率自动调整文件传输带宽
